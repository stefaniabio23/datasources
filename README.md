# datasources

An agent-readable registry of APIs, datasets, and databases.

## What this is, in one sentence

A GitHub-hosted registry of 50+ structured API/dataset/database entries with canonical join keys, MCP gap mapping, and a script-generated Google Doc for sharing.

## Why this exists

Existing public-data directories (`awesome-public-datasets`, `data.gov`, Kaggle) are topic-centric human indexes. They answer "what datasets exist in this category?" but not "given a research question, which dataset should an agent reach for, how does it authenticate, what can it join on, and is there an MCP for it?"

This directory answers the agent question. The two structural pieces nobody else has:

- **`generated/join-key-index.md`** — reverse index of canonical identifiers (`NCT_ID`, `DOI`, `CIK`, `ROR`) mapped to datasets that expose them. Makes cross-source joins planable.
- **`generated/mcp-gap-map.md`** — ranked list of MCP connectors to build, derived from `mcp_status` across the registry. Makes the connector roadmap legible.

## Layout

```
datasources/
  README.md
  SCHEMA.md
  schema/
    entry.schema.yaml
    join-keys.yaml
  entries/
    <domain>/
      <slug>.md
  skills/
    add-dataset-entry/SKILL.md
  scripts/
    validate_entries.py
    generate_shareable.py
    publish_google_doc.py
  generated/
    shareable.md
    shareable.html
    dataset-table.csv
    join-key-index.md
    mcp-gap-map.md
```

## Domains

- `academic/` — scholarly graphs, papers, citations
- `clinical-biotech/` — trials, drugs, regulatory data
- `bio-genomics/` — genes, proteins, expression
- `public-health/` — disease surveillance, indicators
- `healthcare-claims/` — claims, prescribing, utilization
- `finance-markets/` — equities, macro, FX
- `corporate-registry/` — company filings, beneficial ownership
- `news-events/` — global news graphs, event extraction
- `consumer-signal/` — search trends, social, pageviews
- `government-open-data/` — federated open-data portals
- `geospatial/` — maps, weather, biodiversity

## Contributing

The `add-dataset-entry` Skill takes a dataset URL and produces a compliant entry.

1. Run the Skill with the dataset's homepage URL.
2. Skill researches the source, identifies join keys against `schema/join-keys.yaml`, classifies `mcp_status`.
3. Skill writes `entries/<domain>/<slug>.md`.
4. `scripts/validate_entries.py` checks the entry against `entry.schema.yaml`.
5. `scripts/generate_shareable.py` rebuilds the `generated/` outputs.
6. Open a PR with one new entry + regenerated outputs.

See `SCHEMA.md` for the full field spec.

## Distribution

GitHub is canonical. The shareable Google Doc is generated by `scripts/publish_google_doc.py` from `generated/shareable.html` and serves as the EF-facing artifact. It is a downstream rendering, never a source of truth.
